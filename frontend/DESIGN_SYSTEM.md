# MyGo 设计系统

> 面向个人数字游乐场的前端设计与运动语言规范。所有页面和组件应遵循此规范。

## 设计理念

### 风格主轴

热情、灵动、克制而有记忆点的作品集气质。

### 核心特点

- **运动设计优先**: 页面不是静止海报，而是随着进入、滚动、悬停而逐步展开的体验。
- **个性字体优先**: 标题需要强记忆点，正文与辅助信息形成清晰层次，不追求保守稳妥。
- **中性底色 + 橙色焦点**: 以黑、白、灰建立秩序，用橙色注入热度与态度。
- **开放的作品容器**: 页面可以承载文章、作品、实验、角色、视觉叙事或混合体验，不被博客结构束缚。

### 视觉气质

- **更热情**: 不再强调冷峻科技仪器感，而是强调创作者存在感、表达欲和亲近度。
- **更轻盈**: 少用厚重玻璃态，更多使用留白、细边框、简洁块面与字体张力。
- **更鲜明**: 通过字体、动画、导航和 section 节奏建立识别度，而不是依赖复杂材质。

## 色彩系统

### 色彩策略

- **默认支持 light / dark 双主题**
- **主色系**: 中性黑白灰
- **强调色**: 橙色
- **可选辅助色**: 少量冷色仅作功能或实验性点缀，不作为品牌主轴

### 推荐配色

| 角色 | Light | Dark | 用途 |
| ------ | ------- | ------ | ------ |
| 页面背景 | `#E7E5E4` | `#050505` | 页面主背景 |
| 次级背景 | `#F5F5F4` | `#111111` | 区块或浮层 |
| 主文字 | `#171717` | `#FAFAFA` | 大标题、正文 |
| 次要文字 | `#525252` | `#A3A3A3` | 辅助信息 |
| 强调色 | `#EA580C` | `#EA580C` | 链接、按钮、激活态、重点符号 |
| 强调悬停 | `#C2410C` | `#FB923C` | hover / active |
| 边框 | `rgba(234, 88, 12, 0.16)` | `rgba(234, 88, 12, 0.2)` | 卡片、分隔、轻描边 |

### CSS 变量建议

```css
:root {
  --bg-primary: #e7e5e4;
  --bg-secondary: #f5f5f4;
  --text-primary: #171717;
  --text-secondary: #525252;
  --accent: #ea580c;
  --accent-hover: #c2410c;
  --border: rgba(234, 88, 12, 0.16);
}

.dark {
  --bg-primary: #050505;
  --bg-secondary: #111111;
  --text-primary: #fafafa;
  --text-secondary: #a3a3a3;
  --accent: #ea580c;
  --accent-hover: #fb923c;
  --border: rgba(234, 88, 12, 0.2);
}
```

### 使用原则

1. **橙色只用于焦点与节奏点**，不要大面积滥用。
2. **中性色负责秩序**，橙色负责态度。
3. **始终优先使用 CSS 变量或 Tailwind 设计 token**，避免页面内随意硬编码。
4. **允许 light / dark 切换**，但两套主题都必须保持同一套信息层级。

## 字体系统

### 字体策略

- **标题字体**: 使用有明显个性的展示型字体，允许手写感、杂志感或雕塑感。
- **正文字体**: 使用简洁、现代、可读性强的 sans-serif。
- **辅助信息字体**: 可使用 mono 字体增强“作品索引”“标签”“时间轴”“技术栈”等信息层。

### 推荐组合

| 用途 | 字体 | 说明 |
| ------ | ------ | ------ |
| 标题 | `Fredoka` + 中文 sans-serif 回退 | 圆润、厚实、友好，有记忆点但避免幼态化 |
| 正文 | `IBM Plex Sans` | 保持清晰、现代和稳定的阅读体验 |
| 辅助信息 | `IBM Plex Mono` / `ui-monospace` | 用于时间、标签、索引、元信息 |

### 字号建议

| 元素 | 字号 | 行高 | 字间距 |
| ------ | ------ | ------ | -------- |
| 超大标题 | `clamp(44px, 8vw, 108px)` | 0.96-1.02 | -0.04em |
| 主标题 | `clamp(28px, 4vw, 56px)` | 1.02-1.1 | -0.03em |
| section 标题 | `clamp(20px, 2.4vw, 32px)` | 1.1 | -0.02em |
| 正文 | 16px-18px | 1.65-1.8 | 0 |
| 小标签 / 元信息 | 12px-13px | 1.4 | 0.06em |

### 规则

- 不再默认使用 `Fraunces` 作为标题字体。
- 不要使用看起来“太安全”的系统默认字体作为主标题。
- 标题和正文之间必须有明显人格差异。
- 当前默认标题方向应偏圆润、饱满、亲和，而不是锋利冷峻。

## 运动设计

### 动画原则

- **进入即表演**: 首屏需要有明确的入场体验。
- **少而准**: 动效不需要无处不在，但关键时刻必须足够有记忆点。
- **跟随结构**: 动效应强化信息层级，而不是分散注意力。
- **优先平滑、连续、自然**: 避免生硬弹跳和廉价效果。

### 推荐动效模式

#### 首屏动效

- 标题逐字或逐词出现
- loading / 首次进入过渡
- 主导航和 hero 内容错峰进入

#### section 动效

- section 进入时使用 stagger reveal
- 滚动触发淡入、上移、遮罩揭示
- 卡片 hover 可以有轻微位移、旋转或边框高亮

#### 交互动效

- 导航 active 状态过渡明确
- 主题切换需要有清晰但不过分夸张的反馈
- hover 不只是变色，可以结合位移、下划线、生长边框、局部旋转

### 技术建议

- React 项目优先使用 `framer-motion`
- 小型微交互可用 CSS transition / keyframes
- 复杂滚动叙事可按需引入 GSAP / Lenis，但需控制复杂度

## 组件规范

### 导航

- 允许使用固定侧边导航或固定顶部导航
- 导航本身应是风格表达的一部分，而不是普通链接列表
- 当前激活项必须有明确状态差异
- 移动端导航应有完整展开/收起动效

### 卡片

- 更偏轻量边框卡片，而不是厚重玻璃态
- 圆角建议 `12px`-`16px`
- 边框可使用橙色低透明度描边
- hover 可通过边框增强、轻微 lift、轻微 tilt 提升质感

### 按钮

```css
.btn-primary {
  background: var(--accent);
  color: var(--bg-primary);
  border-radius: 10px;
  padding: 12px 22px;
}

.btn-secondary {
  background: transparent;
  color: var(--text-primary);
  border: 1px solid var(--border);
}
```

规则：

- 主按钮数量要少，让重点更明确
- 次要按钮应克制，不要和主按钮争夺注意力

### 标签 / 元信息

- 优先用 mono 字体
- 更像作品索引、技术栈标签、时间节点，而不是可爱徽章
- 颜色以中性色为主，必要时用橙色点亮

### 链接

- 普通链接可保持中性色
- 强调链接使用橙色
- hover 可搭配下划线、偏移动画、颜色加强

## 页面结构

### 推荐结构

```html
<div class="app-shell">
  <nav class="site-nav">...</nav>
  <main>
    <section class="hero">...</section>
    <section class="content-section">...</section>
    <section class="content-section">...</section>
  </main>
</div>
```

### 页面组织原则

- 页面可以是**单页滚动 section**，也可以是**多页面体验入口**
- 首页应更像“作品入口”或“世界观入口”，而不是传统博客封面
- 文章、作品、实验、角色页可以共存，但需要统一的视觉节奏
- 留白、字体层次、动画顺序优先于复杂背景装饰

---

## 不再推荐的旧规则

- 不再要求固定深色碳灰主题
- 不再要求玻璃态成为默认卡片材质
- 不再要求青绿色作为品牌主强调色
- 不再把“中心卡片式页面”视为默认布局
- 不再默认以博客页面结构为主

---

## 开发检查清单

- [ ] 是否支持 light / dark 两套主题且都成立
- [ ] 是否使用了有辨识度的标题字体
- [ ] 是否把动效当作核心体验而不是装饰
- [ ] 是否让导航本身具有设计感
- [ ] 是否控制了橙色的使用面积，让焦点清晰
- [ ] 是否避免回到厚重、冰冷、机械化的旧风格
- [ ] 是否允许页面承载作品、实验、文章、角色等不同媒介形式
- [ ] 装饰元素是否加了 `aria-hidden="true"`
